1.图像分类原理

模型会接收一张图片，然后会输出一组概率，分别是该图片为 Logo 的概率与该图片为其他图片的概率，从而通过概率来判断这张图片是 Logo 类还是 Other 类

1.1感知机

其中输入的图片，将其展开后，可以获得输入 ，而模型可以看做有两个节点，每个节点都会有一个输出，分别代表着对输入为 Logo 和 Other 的判断，但这里的输出暂时还不是概率，只是模型输出的一组数值：

上图这个结构是感知机，中间绿色的节点叫做神经元，是感知机的最基本组成单元。如果有多层（绿色的）神经元的话，我们就称之为多层感知机。

  • 神经元是关于输入的一个线性变换，每一个输入 x 都会有一个对应的权值，上图中的 y 的计算方式为：，权重与偏移项都是学习到的参数，激活函数是一个可选参数。

  • 将和这一组数值转换为概率需要用到softmax，即，且转换后的概率和为1。

  • 有的时候也会使用 sigmoid 激活函数：将1 个数值转换为 0 到 1 之间的概率。

1.2全连接层（nn.Linear）

在卷积神经网络中绿色那一层称为全连接层（Full Connection Layer, fc）。

  • 全连接层的输入也不是原始图片数据，而是经过多层卷积提取的特征。全连接层的输入到是固定的，数目等于最后一层特征图所有元素的数目。

  • 有的网络，如EfficientNet是可以接收任意尺度的输入的，只需要在最后的特征图后面加一个全局平均即可，也就是将每个特征图进行求平均，用平均值（黄色的圈）代替特征图。

1.3卷积神经网络

多层感知机是卷积神经网络的前身，由于自身的缺陷（参数量大、难以训练），使其在历史上有段时间一直是停滞不前。卷积神经网络的最大作用就是提取出输入图片的丰富信息，然后再对接上层的一些应用。

  • 如何验证卷积神经网络是可靠、可用的呢？在业界中有个标杆——ImageNet。

1.3.1 VGG

  • VGG提供了 A-E 6种不同的 VGG 网络（层数不一样）。VGG19 的效果虽说最好，但是综合模型大小等指标，在实际项目中 VGG16 用得更加多一点。

  • VGG贡献：

    • 证明了随着模型深度的增加，模型效果也会越来越好。
    • 使用较小的 3x3 的卷积，代替了 AlexNet 中的 11x11、7x7 以及 5x5 的大卷积核。

  • VGG 中将 5x5 的卷积用 2 层 3x3 的卷积替换；将 7x7 的卷积用 3 层 3x3 的卷积替换。这样做首先可以减少网络的参数，其次是可以在相同感受野的前提下，加深网络的层数，从而提取出更加多样的非线性信息。

减少参数：例如，用两个3x3的卷积核替代一个5x5的卷积核，可以将参数量减少为(3x3)x2=18，而5x5的卷积核有25个参数。

相同感受野：假设输入特征图的尺寸为H x W，经过一个步长为1和填充为1的3x3卷积操作后，输出的特征图尺寸仍然是H x W。两个这样的3x3卷积层堆叠在一起，感受野大小仍然是3x3，但网络的深度增加了。通过这种方式，VGG网络可以增加网络的深度，同时保持感受野的一致性。

在VGG网络的设计中，每个卷积层后面都紧跟一个池化层（2x2），会导致特征图的尺寸减半，感受野增大。但在讨论感受野保持一致的情况下，主要考虑的是卷积层的操作。然而，一些最新的网络架构，如ResNet和Inception等，采用了其他的下采样方法来增大感受野，如跳跃连接和卷积步长（stride）操作，而不是传统的池化操作。

1.3.2 GoogLeNet

  • GoogLeNet 的核心是 Inception 模块。这个时期的 Inception 模块是 v1 版本，后续还有 v2、v3 以及 v4 版本。研究人员发现，对于同一个类别的图片，主要物体在不同图片中，所占的区域大小均有不同：

  • 如果使用 AlexNet 或者 VGG 中标准的卷积的话，每一层只能以相同的尺寸的卷积核来提取图片中的特征。但是很可能物体以不同的尺寸出现在图片中，那么能否以不同尺度的卷积来提取不同的特征呢？沿着这个想法，Inception 模块应运而生，如下图示：

  • 这里是将原来的相同尺寸卷积提取特征的方式拆分为，使用 1x1、3x3、5x5 以及 3x3 的 max pooling 同时进行特征提取，然后再合并到一起。这样就做到了以多尺度的方式提取图片中的特征。为了降低网络的计算成本，将上述的 Inception 模块做了一步改进，在 3x3、5x5 之前与 pooling 之后添加了 1x1 卷积用来降维，从而获得了 Inception 模块的最终形态。

面试会问，为什么采用 1x1 的卷积或者 1x1 卷积的作用？

1x1 卷积的作用就是用来升维或者降维（channel的维度）。由于 1×1 并不会改变 height 和 width，而最终的channel数由卷积核的数量（filters）决定：若filters>原来的channel数，则升维；反之，降维。
